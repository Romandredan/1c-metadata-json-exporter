# Как внести вклад

Спасибо за интерес к проекту!

## Workflow

1. Сделайте fork репозитория.
2. Создайте ветку: `git checkout -b feature/my-feature`.
3. Внесите изменения, соблюдая стандарты кодирования 1С.
4. Если меняется формат JSON — добавьте пример в `examples/`.
5. Обновите `CHANGELOG.md` в разделе **[Не выпущено]**.
6. Создайте Pull Request с описанием изменений.

## Стандарты кода

- Имена идентификаторов и комментарии — на русском языке
- Каждая экспортная процедура/функция должна иметь описание-комментарий по стандарту 1С №453
- Используйте Попытка/Исключение с записью в журнал регистрации; немое подавление допустимо только для проверки наличия свойства/коллекции у объекта метаданных (полиморфизм платформы)
- Без зависимости от прикладного кода БСП. Регистрация в подсистеме БСП «ДополнительныеОтчётыИОбработки» через стандартный интерфейс `СведенияОВнешнейОбработке()` — штатный сценарий, ломать его при рефакторинге нельзя
