# История изменений

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект следует [семантическому версионированию](https://semver.org/lang/ru/).

## [Не выпущено]

### Планируется
- Блок Связи (регистраторы, владельцы, журналы)

### Добавлено
- Блок `СвойстваКонфигурации` в корне выгрузки (флаг `ВключатьСвойстваКонфигурации`, по умолчанию включён): режим совместимости, режим совместимости интерфейса, основной режим запуска, режим модальности, режим синхронных вызовов, режим управления блокировкой, вариант встроенного языка, основной язык, синоним, поставщик. В режиме отдельных файлов дублируется в каждый файл.
- Подблок `ПодключённыеРасширения` в `СвойстваКонфигурации` (флаг `ВключатьРасширения`, по умолчанию выключен, выведен на форму обработки): имя, синоним, версия, активность. Требует прав; при их отсутствии не добавляется.
- Блок `СвойстваОбъекта` (флаг `ВключатьСвойстваОбъекта`, по умолчанию включён) — свойства, зависящие от вида: справочник (иерархия, длина кода/наименования, владельцы), документ (проведение, нумерация, основания для ввода), регистр накопления (`ВидРегистра` Остатки/Обороты), регистр сведений (периодичность, режим записи), регистр бухгалтерии (корреспонденция, план счетов), регистр расчёта (план видов расчёта), ПВХ (тип значения характеристик), план счетов (макс. субконто). Для регистров — `Регистраторы` (документы-регистраторы, собираются обратным обходом).
- Тип значения константы: для `ВидОбъекта = "Константа"` теперь выгружаются `Типы` и квалификаторы (`Длина`, `Точность`, `ДлинаСтроки`). Раньше описание константы было фактически пустым.
- Блок `ЗначенияПеречисления` для перечислений — массив `{Имя, Синоним}`. Синонимы значений (несущие бизнес-смысл) раньше терялись. Выгружается всегда, независимо от `ВключатьПредопределенные`.
- Маркер обрыва по глубине `{ПолноеИмя, ОбрывПоГлубине: true}` в блоке `Раскрытие`. Позволяет LLM отличить обрыв по пределу `ГлубинаВложенности` от свёртки рекурсии (раньше при достижении предела блок `Раскрытие` просто отсутствовал).
- Константы доступны для выбора в форме выбора метаданных (переключатель «Константы», по умолчанию выключен).

### Исправлено
- В режиме «отдельные файлы» (`ОтдельныеФайлы = Истина`) флаг `ДобавлятьВременнуюМетку` игнорировался: имя файла формировалось напрямую из полного имени объекта без постфикса `_<yyyyMMdd_HHmmss>`. Теперь метка добавляется и здесь (вычисляется один раз на выгрузку, у всех файлов запуска одинаковая). Логика метки вынесена в общую утилиту `СуффиксВременнойМетки`.
- Константы нельзя было выбрать в форме выбора метаданных: коллекция `Метаданные.Константы` отсутствовала в перечне видов формы (атрибут-флаг, переключатель и строка в `ПолучитьКоллекции`). Ядро константу описывать умело, но добраться до неё через интерфейс было нельзя.

### Изменено
- **BREAKING:** значения перечислений больше не попадают в массив `Предопределённые`. Состав перечисления теперь только в блоке `ЗначенияПеречисления`. Для видов Справочник/ПВХ/ПС/ПВР поведение `Предопределённые` не изменилось.

### Внутреннее
- Версия обработки поднята до `2.0.0` (следствие BREAKING-изменения формата).
- Рефакторинг `ОписаниеОбъекта`: специфика вида вынесена в наполнители `ДополнитьОписаниеРеквизитамиИТЧ` и `ДополнитьОписаниеСпецификойВида` (снижение цикломатической сложности, подготовка к блокам обогащения).
- Логика «тип + квалификаторы» вынесена в общую процедуру `ДополнитьОписаниеТипом` (переиспользуется реквизитами и константой).

## [1.0.0] — 2026-05-27

### Добавлено
- Универсальная выгрузка справочников, документов, регистров, перечислений, ПВХ, ПВР, планов счетов
- Три режима: один объект / список / подсистема
- Программный API
- Раскрытие ссылочных типов с защитой от рекурсии
- Опция компактного JSON
- Регистрация как дополнительная обработка БСП
