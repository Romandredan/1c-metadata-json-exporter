# Roadmap обогащения выгрузки

Живой трекер развития формата. Источник идей — `doc/Proposal for evolution.md`; здесь — статус и приоритеты. Обновлять при закрытии каждого блока.

Легенда реализуемости (внешняя обработка в режиме 1С:Предприятия):
- ✅ доступно через свойства объектов метаданных;
- ⚠️ доступно с оговорками (права, производительность, чтение данных);
- ❌ нереализуемо обработкой в режиме Предприятия.

## Процесс работы

- Каждый блок — отдельная ветка `feature/<имя>` от master + PR. В master напрямую не коммитим.
- При изменении формата синхронно править: код `ObjectModule.bsl` + `schema/metadata-output.schema.json` + `doc/format.md` + `CHANGELOG.md` (+ перегенерация `examples/` при необходимости).
- Имена свойств объектов метаданных выверять через справочник платформы (`ОбъектМетаданных: <Вид>`) перед использованием.
- Breaking-изменения формата помечать `BREAKING` в `CHANGELOG.md` и поднимать мажорную версию.
